## Build & Run

- Open `WhatsTheScore?.xcodeproj` in Xcode and build/run (Cmd+R)
- iOS deployment target: 16.5
- Swift 5.0, SwiftUI-only (no UIKit)
- Single SPM dependency: `firebase-ios-sdk` (FirebaseAuth + FirebaseFirestore)
- Bundle ID: `com.VPTechnology.WhatsTheScore-`
- Requires "Sign in with Apple" capability in Xcode
- Firebase config lives in `GoogleService-Info.plist` at the project root
- No test targets exist yet

## Architecture

MVVM pattern with Firebase as the sole backend. No local persistence — all data flows through Cloud Firestore with offline caching enabled (100MB `PersistentCacheSettings`).

### Layers

**Models** (`WhatsTheScore?/Models/`): Plain `Codable` structs — `AppUser`, `Leaderboard` (with embedded `LeaderboardMember` array), `Match` (with `PlayerResult` array), `Rank`/`RankTier`, `PointSystem`. The `Leaderboard` model has a custom `Codable` init for backwards compatibility with the `memberIds` field.

**Services** (`WhatsTheScore?/Services/`): Singleton Firebase wrappers accessed via `.shared`. `AuthService` handles Sign in with Apple → Firebase Auth credential exchange. `LeaderboardService` and `MatchService` handle all Firestore CRUD. Services use both one-shot `getDocument`/`getDocuments` and real-time `addSnapshotListener`.

**ViewModels** (`WhatsTheScore?/ViewModels/`): `@MainActor ObservableObject` classes. `AuthViewModel` is the root — injected as `@EnvironmentObject` from `WhatsTheScore_App`. Other VMs (`HomeViewModel`, `LeaderboardViewModel`, `CreateMatchViewModel`) are created per-view.

**Views** (`WhatsTheScore?/Views/`): Organized by feature — `Auth/`, `Home/`, `Leaderboard/`, `Match/`, `Profile/`.

**Theme** (`WhatsTheScore?/Theme/AppTheme.swift`): `RankTheme` provides rank-tier gradient colors and position colors. `CardStyle` ViewModifier and `RankProgressInfo` helper for rank progress bars.

### Firestore Data Model

```
users/{uid}           → AppUser (leaderboardIds array)
leaderboards/{id}     → Leaderboard (members array embedded, memberIds for security rules)
  /matches/{id}       → Match (players array with placements and points)
```

Matches are a sub-collection under leaderboards. Member data (points, wins, gamesPlayed) is denormalized into the `Leaderboard.members` array — updates happen via read-modify-write in `LeaderboardService.updateMemberPoints`. Security rules are in `firestore.rules`.

### Ranking System

8 tiers (Iron → Immortal) × 3 divisions = 24 ranks. Each division spans 100 points, each tier spans 300. Calculated from points via `Rank.fromPoints()`. Points can go negative (displays as Iron 1). Point systems are defined in `PointSystem.presets(forPlayerCount:)` for 2-4 players.

### Key Patterns

- Invite codes: 6-character alphanumeric, generated by `InviteCodeGenerator`, used to join leaderboards
- Firestore `in` queries limited to 30 items — `LeaderboardService` chunks `leaderboardIds` accordingly
- `ContentView.swift` and `Persistence.swift` are unused stubs
